# 通信プロトコル仕様書 — 東京Bro（喧嘩番長-Crash Battle-）

## 通信機能概要

本作「喧嘩番長-Crash Battle-」はスマートフォン向けオンラインゲームであり、以下の通信機能を実装している。

- クラウドセーブ（プレイヤーデータの自動サーバー保存）
- ガチャ・課金処理（サーバーサイドでの確率処理）
- 最大4人リアルタイム協力バトル
- ランキング機能
- マスターデータ更新（バージョン管理）

---

## 通信仕様概要

### 使用プロトコル
- **REST API**：クライアント→サーバーへのデータ送受信（セーブ同期・ガチャ・ランキング）
- **WebSocket（またはHTTP Long Polling）**：リアルタイム協力バトルの同期通信
- **HTTPS**：全通信をSSL/TLS暗号化（証明書はサーバー側で管理）

### 接続情報
| 項目 | 値 |
|------|-----|
| APIベースURL | https://api.kenkabanchou-crash.spikechunsoft.jp/v1/（推定） |
| 通信ポート | 443（HTTPS標準）|
| タイムアウト | 10秒（APIリクエスト共通）|
| リトライ回数 | 最大3回（指数バックオフ）|

## リアルタイム協力バトル通信仕様

### 接続方式
WebSocketを使用したリアルタイム双方向通信（サーバー：スパイク・チュンソフト管理）

### メッセージフォーマット（JSON）

**プレイヤーアクション送信：**
```json
{
  "type": "PULL_ACTION",
  "player_id": "uuid",
  "direction_x": 0.87,
  "direction_y": 0.50,
  "pull_distance": 0.75,
  "timestamp": 1413763200000
}
```

**バトル状態同期（サーバー→全クライアント）：**
```json
{
  "type": "BATTLE_STATE_UPDATE",
  "turn": 5,
  "enemies": [
    {"id": 1, "hp": 15000, "max_hp": 25000}
  ],
  "players": [
    {"player_id": "uuid1", "hp": 3200, "max_hp": 5000}
  ]
}
```

### 同期遅延許容値
- 理想値：遅延50ms以内
- 許容値：遅延200ms以内（200ms超でラグ補正処理発動）
- 切断判定：5秒間通信途絶で自動的に同期切断・CPU代替処理に切替

## オンラインランキング

### ランキング種別
| ランキング名 | 集計単位 | 集計指標 |
|------------|---------|---------|
| バトルスコアランキング | 週間リセット | クエストバトルの累計獲得スコア |
| 協力バトルランキング | 月間リセット | 協力バトルでのダメージ総量 |
| 男気ランキング | 常時更新 | 現在の男気ゲージ値 |

### ランキングデータ取得API
- GET /ranking/{type}?page={n}&per_page=100
- レスポンス：プレイヤー名・スコア・順位・アバター情報のリスト（100件単位）
