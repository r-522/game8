# 技術設計書 — 喧嘩番長5 漢の法則

## 対象ハードウェア仕様（PlayStation Portable）

| 項目 | 仕様 |
|---|---|
| CPU | MIPS R4000ベース 333MHz（PSP-2000以降：フル使用可） |
| GPU | カスタムGPU（最大664万ポリゴン/秒描画能力） |
| RAM | 32MB（PSP-1000） / 64MB（PSP-2000以降） |
| VRAM | 4MB（メインRAM共有） |
| ストレージ | UMD光ディスク（最大1.8GB）+ メモリースティック Duo |
| 解像度 | 480×272ピクセル（16:9） |
| フレームレート目標 | 30fps（フィールド・バトル共通） |
| サウンド | ステレオ出力（AT3/MP3形式対応） |
| 通信 | IEEE 802.11b（PSPアドホック通信） |
| バッテリー | 1800mAh（PSP-2000標準）。動作時間約4〜5時間 |

## システムアーキテクチャ概要

```
┌─────────────────────────────────────────────┐
│                Application Layer                │
│  GameMain / SceneManager / EventSystem          │
├─────────────────────────────────────────────┤
│              Game Logic Layer                   │

│  SaveSystem / ItemSystem / QuestManager         │
├─────────────────────────────────────────────┤

│  3DRenderer / UIRenderer / ParticleSystem       │
│  AnimationSystem / CameraSystem                 │
├─────────────────────────────────────────────┤
│              Platform Layer                     │

```

### モジュール間通信方針
- ゲームロジック層はメッセージキュー方式でレンダリング層に描画指示を渡す
- 直接参照はゲームロジック→レンダリングの一方向のみ（循環参照禁止）
- セーブシステムはシリアライズ専用スレッドで非同期処理

## レンダリングシステム

### ポリゴン数制限（設計目標値）

| 対象 | ポリゴン数上限 | 備考 |
|---|---|---|
| 主人公キャラクター | 3,500 ポリゴン | LOD2段階：近距離3500/遠距離1500 |
| ボスキャラクター | 4,000 ポリゴン | カットシーン専用モデル：6000 |
| 雑魚敵キャラクター | 1,500〜2,000 ポリゴン | 同時最大5体を想定 |
| NPCキャラクター | 800〜1,200 ポリゴン | 視野外はカリングで非描画 |
| 背景ステージ（フィールド） | 10,000〜15,000 ポリゴン | フラスタムカリングで実効低減 |
| バトル背景 | 5,000〜8,000 ポリゴン | 個別シーン用 |

### テクスチャ仕様

| 種別 | 解像度 | フォーマット |
|---|---|---|
| キャラクターテクスチャ | 256×256 | TIM2（RGBA4444） |
| 顔テクスチャ | 128×128 | TIM2（RGBA8888） |
| 背景テクスチャ | 512×256 | TIM2（RGB565） |
| UIテクスチャ | 画面幅に合わせ最大512×256 | TIM2（RGBA8888） |
| エフェクトテクスチャ | 64×64〜128×128 | TIM2（RGBA4444） |

### フレームレート管理
- 目標30fps（33.3msec/frame）。VSync有効
- フィールド描画：ロジック16ms + 描画14ms = 計30ms以内
- バトル描画：ロジック10ms + 描画20ms = 計30ms以内
- ロード中はロゴ表示で補完（非同期ストリーミングロード）

## 物理演算・衝突判定

### 衝突判定方式
PSPのGPU機能を使った物理演算は行わず、CPU側で簡易衝突判定を実装する。

- **キャラクター間衝突：** 球体バウンディングボリューム（半径1.0〜1.5m）
- **フィールドコリジョン：** 矩形コリジョンメッシュ（見えないポリゴンとして配置）
- **攻撃判定：** 技発動フレームのみ矩形ヒットボックスを有効化
- **判定レート：** 1フレームごと（30fps = 毎30ms判定）

### バトル判定タイミング
ボタン入力に対して以下のタイムウィンドウを設ける：
- PERFECT：±2フレーム（±66ms）
- GREAT：±4フレーム（±133ms）
- GOOD：±6フレーム（±200ms）
- BAD：±8フレーム（±266ms）
- MISS：それ以外

## メモリ管理方針

### メモリ配分（PSP-2000 64MB想定）

| 用途 | 割り当て |
|---|---|
| ゲームロジック・コード | 4MB |
| フィールドステージデータ | 8MB（ストリーミングで入替） |
| キャラクターモデル（最大5体分） | 10MB |
| テクスチャVRAMミラー | 4MB |
| BGM（AT3ストリーミングバッファ） | 4MB |
| SEバンク（メモリ常駐） | 3MB |
| UI・スプライト | 2MB |
| セーブデータ作業領域 | 1MB |
| スタック・ヒープ | 8MB |
| 予備・安全マージン | 20MB |

### ストリーミング戦略
- BGMはAT3形式でUMDからストリーミング再生（メモリへの全展開不要）
- フィールドは地区単位でストリーミングロード。地区間移動時に非同期切り替え
- キャラクターモデルはバトル開始前にプリロード、バトル終了後に解放

## AI実装方針

### 雑魚敵AI
有限状態機械（FSM）で実装。状態数を最小限に抑えメモリ・CPU負荷を削減。

```
状態：[待機] → [接近] → [攻撃] → [のけぞり] → [復帰] → [待機]
```

- 接近状態：プレイヤーが半径5m以内に入ると遷移
- 攻撃状態：プレイヤーが半径2m以内で攻撃コマンド選択（ランダム比率）
- のけぞり状態：被ダメージ時に強制遷移。0.5秒で復帰

### ボスAI
雑魚敵FSMに加え、HPフェーズ制の行動変化ロジックを実装。

- フェーズ1（HP100〜60%）：通常パターン
- フェーズ2（HP60〜30%）：攻撃頻度増加、特技使用
- フェーズ3（HP30〜0%）：超気合技解放、攻撃力強化

## サウンドシステム

### 音声フォーマット
- BGM：AT3（ATRAC3plus）形式、128kbps、ステレオ
- SE：VAG形式（8bit PCM）、モノラル、44.1kHz
- ボイス：AT3形式、64kbps、モノラル

### サウンドチャンネル管理
PSPのハードウェアサウンドチャンネルは最大24本。以下の配分で管理する：
- BGM：2チャンネル（ステレオ）
- SE同時再生：最大8チャンネル
- ボイス：2チャンネル（主人公+相手）
- アンビエント（環境音）：2チャンネル
- 予備：10チャンネル

## ロード最適化

### ロード時間目標
- 初回起動〜タイトル表示：最大15秒
- セーブデータロード〜フィールド表示：最大8秒
- 地区間移動：最大5秒（ローディング画面表示）
- バトル突入：最大3秒（シームレス演出で補完）

### 最適化手法
- テクスチャをTIM2形式に統一しGPU転送を最適化
- ステージデータをセルグリッド分割し表示範囲外のデータをアンロード
- セーブデータはファイルサイズを最小化（バイナリ形式・圧縮なし）し書き込み時間を短縮
- UMDシーク回数削減のためデータをファイル内で地区ごとに連続配置
