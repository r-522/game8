# 通信プロトコル仕様書 — 喧嘩番長

## 本作品の通信機能について

**喧嘩番長（2005年、PlayStation 2版）は、シングルプレイ専用タイトルである。**

PlayStation 2はネットワークアダプタ（HDD内蔵型または外付け）を使用したオンライン接続に対応していたが、本作においてはオンライン機能・ローカル対戦機能・通信機能は一切実装されていない。本作は「番長としての男気の追求」という一人用のアドベンチャー体験を中心に設計されており、通信対戦を盛り込む設計意図は企画段階からなかった。

---

## 通信機能なし（シングルプレイ専用）の根拠

| 項目 | 詳細 |
|------|------|
| 対戦モード | なし |
| オンラインランキング | なし |
| ダウンロードコンテンツ | なし |
| PS2ネットワークアダプタ対応 | 非対応 |
| USB/i.LINKを使った通信 | 非対応 |
| セーブデータ共有 | メモリーカードの物理的な受け渡しのみ（通信機能外） |

## 将来的な通信機能拡張案（仮想設計）

本項では、もし本作にシリーズ展開や後継作において通信機能を追加するとした場合の設計案を記述する（実際には実装なし）。

### 案1: PSP版移植時のアドホック対戦モード

**想定機能**: PSP版への移植（実際には2006年に喧嘩番長2 for PSPが発売された）を想定した場合のローカル対戦。

#### 接続手順（アドホック通信）
```
1. ゲームメニュー「対戦モード」を選択
2. 「アドホック対戦」選択
3. ホスト側: 「待ち受け」を選択 → PSPがアドホックモードでアクセスポイントとして動作
4. ゲスト側: 「接続」を選択 → 近くのホストを検索・一覧表示
5. ゲストがホストを選択 → 接続リクエスト送信
6. ホストが承認 → 対戦準備画面へ
```

#### 対戦モード案
- **1対1番長対決**: プレイヤーキャラクター同士がメンチ→タンカ→戦闘のフルフローで対戦
- **タンカ対決のみ**: タンカバトルの速度・正確さを競うミニゲーム的対戦
- **男気競い**: 一定時間のフィールドプレイの男気増減結果を競う

#### 対戦用データ交換仕様
| データ項目 | 送信サイズ | 頻度 |
|-----------|----------|------|
| キャラクター座標 | 12バイト（x,y,z各4バイト） | 毎フレーム（30fps） |
| キャラクター向き | 4バイト（float） | 毎フレーム |
| 行動状態フラグ | 2バイト | 状態変化時 |
| ダメージイベント | 4バイト（攻撃者ID + ダメージ量） | ヒット発生時 |
| タンカ入力 | 1バイト（ボタン番号） | 入力時 |

#### 帯域要件
- PSPアドホック通信: IEEE 802.11b（最大11 Mbps）
- 対戦1セッションの帯域: 推定30〜50 Kbps（2人対戦時）

### 案2: 後継作におけるオンラインランキング

**想定機能**: シリーズ後継作（PlayStation 3/PlayStation Network対応）での全国ランキング投稿。

#### ランキング項目案
| ランキング種別 | 集計対象 | 更新頻度 |
|-------------|---------|---------|
| 最高男気ランキング | プレイ中の最高男気値 | リアルタイム（戦績投稿時） |
| 最速クリアタイム | ストーリークリアまでのプレイ時間 | ゲームクリア後に投稿 |
| コンボ数ランキング | 1戦闘での最大コンボ数 | 戦闘終了後に投稿 |
| タンカ成功率 | 全タンカの成功率（%） | 投稿時に集計 |

#### 通信プロトコル案（HTTP REST）
```
POST /api/ranking/submit
Content-Type: application/json

{
  "userId": "PSN_USER_ID",
  "rankingType": "OTOKOGI_MAX",
  "value": 98,
  "timestamp": "2026-01-01T00:00:00Z",
  "checksum": "HMAC_SHA256_VALUE"
}

Response:
{
  "rank": 42,
  "totalEntries": 15000,
  "nearbyRanks": [...]
}
```

### 案3: セーブデータQRコード共有（アナログ通信）

実際にPS2時代のゲームで使用されていたアナログな「通信」手法として、ゲーム雑誌や攻略本に掲載されたセーブデータのQRコード・数値コードをゲーム内で入力することで特典を受け取る仕組みが考えられる。

- **入力方法**: ゲームメニューの「シークレットコード入力」画面で10桁の数値を入力
- **用途**: 隠し技の解放、限定コスチュームの入手、伝説アルバムのヒント表示
- **セキュリティ**: 単純なチェックサムによる不正入力防止
