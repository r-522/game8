# 技術設計書 — 喧嘩番長

## 対象ハードウェア仕様（PlayStation 2）

| 項目 | 仕様 |
|------|------|
| CPU | Emotion Engine (MIPS R5900系) 294.912 MHz |
| GPU | Graphics Synthesizer 147.456 MHz |
| メインRAM | 32 MB RDRAM |
| ビデオRAM | 4 MB eDRAM (VRAM) |
| サウンドプロセッサ | IOP (I/O Processor) + SPU2 |
| サウンドRAM | 2 MB |
| メディア | DVD-ROM（最大8.54 GB） |
| 最大解像度 | 640×480 (480i), 640×480p (480p対応) |
| カラーデプス | 32bpp |
| ポリゴン性能（理論値） | 約7500万ポリゴン/秒（テクスチャなし） |
| 実用ポリゴン数 | 約50〜150万ポリゴン/秒（テクスチャ・エフェクト込み） |
| フレームレート目標 | 30 FPS（通常時）、戦闘時も30 FPS維持を目標 |

## システムアーキテクチャ概要

### 全体構成

```
[ゲームループ（30FPS固定）]
       │
       ├── [入力管理モジュール]
       │       L DualShock2 入力サンプリング
       │
       ├── [ゲーム状態管理]（FSM: Finite State Machine）
       │       L フィールド状態 / メンチ状態 / タンカ状態 / 戦闘状態 / イベント状態
       │
       ├── [キャラクター管理]
       │       L プレイヤー制御 / 敵AI / NPC
       │
       ├── [戦闘システム]
       │       L コンボ計算 / ダメージ計算 / ヒット判定
       │
       ├── [マップ・ステージ管理]
       │       L ストリーミングロード / コリジョン
       │
       ├── [男気・パラメータ管理]
       │       L イベント駆動で更新
       │
       ├── [レンダリングパイプライン]
       │       L 3Dシーン描画 / HUD描画 / エフェクト
       │
       └── [サウンドシステム]
               L BGM再生 / SE再生 / ボイス管理
```

### 状態遷移マネージャー（FSM）
ゲーム全体の状態を`enum GameState`で管理し、遷移条件に応じてモジュール切り替えを行う。状態ごとに対応するUpdate/Drawサブルーチンが呼ばれる設計とする。

## レンダリングシステム

### ポリゴン数制限

| オブジェクト種別 | ポリゴン数上限（推定） |
|---------------|-------------------|
| プレイヤーキャラクター | 3,000〜4,000ポリゴン |
| 主要ボスキャラクター | 3,500〜4,500ポリゴン |
| ザコ敵（同時表示5体上限） | 500〜1,000ポリゴン/体 |
| NPC（通行人） | 300〜500ポリゴン/体 |
| 建物・背景（1ブロック） | 10,000〜20,000ポリゴン |
| 総シーン描画ポリゴン上限（推定） | 約100,000〜150,000ポリゴン/フレーム |

### テクスチャ仕様
- 主要キャラクター顔テクスチャ: 256×256ピクセル（32bpp）
- 体テクスチャ: 256×256ピクセル
- 背景テクスチャ: 512×256または256×256（タイリング使用）
- UI・HUDテクスチャ: 128×128以下に抑える
- 全テクスチャの合計VRAMは4MB以内に収める（テクスチャスワッピング管理必須）

### ライティング
- キャラクターへのダイナミックライティング: 方向光（Directional Light）1基
- 環境光（Ambient Light）による底上げ
- 昭和ヤンキー世界観に合わせた暖色系ライティング（夕方・夜は赤橙〜青紫のカラーグレーディング）

## 物理演算・衝突判定

### 衝突判定方式
PlayStation 2のEmotion Engineを活用した独自衝突判定ライブラリを使用（ミドルウェア依存を最小化）。

- キャラクター間: カプセル型コライダー同士の交差判定
- キャラクターと地面: 高さマップ（Height Map）ベースの地面判定
- キャラクターと壁: ボックス型コライダーによる押し戻し

### 攻撃ヒット判定
- 攻撃モーション中に「アクティブヒットボックス」を一時的に生成
- フレーム単位でヒットボックスのキャラクター間交差をチェック
- 1フレームあたりの判定処理はEEのVU（Vector Unit）で並列計算

## メモリ管理方針（RAM 32MB制限への対応）

| メモリ領域 | 割当容量（推定） | 用途 |
|-----------|--------------|------|
| メインプログラム | 約6 MB | ゲームロジック全体 |
| キャラクターデータ（動的ロード） | 約5 MB | 表示中のキャラメッシュ・アニメーション |
| ステージデータ（ストリーミング） | 約8 MB | 現在エリアの背景・コリジョン |
| テクスチャデータ | 約4 MB | VRAM転送用バッファ |
| サウンドデータ | 約3 MB | BGM・SE・ボイスバッファ |
| システム予約 | 約2 MB | OS・I/Oバッファ |
| 残余バッファ | 約4 MB | 一時使用領域・キャッシュ |

### ストリーミングロード方針
- フィールド移動時に次エリアのデータをバックグラウンドでDVDからロード
- 駅間の電車移動中にロードを完了させる設計（ユーザーへのロード待ち見せ最小化）
- 使用していないエリアデータはメモリから解放（ダブルバッファリング）

## AI実装方針

### 敵AIの行動ステート

```
[待機] → [発見（メンチ範囲内）] → [メンチ返し] → [戦闘] → [怯み] → [回復] → [戦闘]
                                                              └→ [敗北]
```

- 各ステートはシンプルな条件分岐で実装（フルゲームAIは不使用）
- 敵の行動はEmotion EngineのIOP（サブCPU）側で非同期処理
- ボス専用のAIは独自のパターンテーブルを参照

### 群衆制御
- ザコ敵の群れ行動: 先頭敵がプレイヤーに接近、残りは周囲を包囲する簡易ステアリング
- 同時戦闘上限5体（処理負荷軽減のため）

---

## サウンドシステム

### 構成
- BGM: ADPCM圧縮ストリーミング（SPU2使用）、ループポイント指定あり
- SE: 短ループファイル（VAGフォーマット）、最大24チャンネル同時再生
- ボイス: シーン再生時のみストリーミング（常駐不可）

### ボイス管理
- フルボイス実装。セリフは専用サウンドバンクに収録
- 戦闘中のかけ声・うめき声はランダム再生（複数バリエーション）

---

## ロード最適化

### 対策一覧
- DVDシーク時間最小化のためデータ配置をディスク内側から重要度順に配置
- よく使うテクスチャ（UI・主人公モデル）はメモリ常駐
- ステージデータはゾーン単位に分割し、必要分だけロード
- イベントムービーはDVDからリアルタイムストリーミング再生（メモリ節約）
- ロード画面（電車移動演出）を活用してプレイヤーに待ちを感じさせない
