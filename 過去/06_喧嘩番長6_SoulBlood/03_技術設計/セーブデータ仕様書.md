# セーブデータ仕様書 — 喧嘩番長6 Soul & Blood

## セーブデータ構造定義

### セーブデータ全体構造

| フィールド名 | 型 | サイズ（バイト） | 説明 |
|------------|-----|--------------|------|
| save_version | uint16 | 2 | セーブデータバージョン番号（互換性管理） |
| checksum | uint32 | 4 | データ破損検出用チェックサム（CRC32） |
| save_time | int64 | 8 | セーブ日時（UNIX時間） |
| play_time_sec | uint32 | 4 | 累計プレイ時間（秒） |
| player_data | PlayerSaveData | 可変 | プレイヤーキャラクターデータ |
| story_flags | StoryFlags | 64 | ストーリー進行フラグ集合 |
| collection_data | CollectionData | 可変 | 収集要素データ |
| otokogi_data | OtokogiSaveData | 8 | シブシャバ度データ |
| world_state | WorldStateSave | 可変 | フィールド状態データ |
| dojo_data | DojoSaveData | 可変 | アジトデータ |
| **合計推定** | — | **約 4〜6 KB** | 1スロット分 |

## 保存タイミング

### 手動セーブ
- ポーズメニュー → セーブ → スロット選択
- セーブスロット数：3スロット
- 上書き確認ダイアログあり

### 自動セーブ
- 戦闘終了時（勝利時のみ）
- ストーリーイベント完了時
- 学期が進んだ時（章移行時）
- 自動セーブはスロット0（専用の自動セーブスロット）に保存

---

## ストレージ使用量

| 項目 | 容量 |
|------|------|
| 手動セーブ（1スロット） | 約6KB |
| 手動セーブ（3スロット合計） | 約18KB |
| 自動セーブ（1スロット） | 約6KB |
| **合計最大使用量** | **約24KB** |
| システムデータ（設定等） | 約2KB |
| **総合計** | **約26KB** |

3DSのSDカードまたは内部ストレージに保存。必要容量は非常に少なく、空き容量不足のエラーはほぼ発生しない設計。

---

## セーブスロット数

| スロット | 種別 | 説明 |
|---------|------|------|
| スロット1 | 手動 | プレイヤーが任意に保存 |
| スロット2 | 手動 | プレイヤーが任意に保存 |
| スロット3 | 手動 | プレイヤーが任意に保存 |
| スロット0（AUTO） | 自動 | システムが自動保存（上書き専用） |

### 複数セーブの活用
- 分岐点（片桐戦前等）でスロット分岐保存を推奨する画面表示あり
- スロット間のコピー機能は未搭載（シリーズ慣行に準じ単純設計）
