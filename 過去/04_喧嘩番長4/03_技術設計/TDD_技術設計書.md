## システムアーキテクチャ概要

本作はバレット社内製ゲームエンジン（仮称：BulletEngine v3）を使用。PSP向けに最適化されたC++コードベースで構築される。

```
[Application Layer]
  GameMain.cpp  ← ゲームループ管理・シーン遷移
       |
[Scene Layer]
  FieldScene / CombatScene / MenchiScene / TankaScene / UIScene
       |
[System Layer]
  GameSystems:
    - RenderSystem（3D描画）
    - CombatSystem（戦闘処理）
    - MenchiSystem（メンチ対決）
    - TankaSystem（タンカリズムゲーム）
    - AISystem（敵AI）
    - AudioSystem（BGM/SE）
    - InputSystem（パッド入力）
    - SaveSystem（セーブ/ロード）
       |
[Engine Layer]
  BulletEngine v3:
    - ResourceManager（UMDからのアセットロード）
    - MemoryManager（固定メモリプール）
    - GraphicsCore（GE描画コマンド生成）
    - PhysicsCore（簡易コリジョン）
```

## 物理演算・衝突判定

PSPのCPUパワーの制約上、フルフィジックスエンジンは使用せず独自の軽量衝突判定を実装する。

### 衝突判定方式

| 種別 | 方式 | 備考 |
|-----|------|------|
| キャラクター同士 | AABB（軸並行境界ボックス）+ カプセルコライダー | ヒットバックの方向計算に使用 |
| 地形との衝突 | レイキャスト（高さマップ）+ メッシュコライダー（単純化） | 落下判定・歩行面検出 |
| 攻撃ヒット判定 | 球体コライダー（ヒットスフィア） | フレーム単位でのオン/オフ制御 |
| カメラ | レイキャスト（壁貫通防止） | カメラが壁に埋まらないよう補正 |

### 物理パラメータ
- 重力加速度：9.8 m/s²（実際の体感と合わせて調整）
- キャラクターの高さ：カプセル高さ1.7m、半径0.3m
- ヒットバック距離：攻撃力に応じて0.5m〜2.0m

## メモリ管理方針

PSP-1000のRAM制限（32MB）への対応を最優先に設計する。

### メモリ割り当て

| 領域 | 割り当て容量 | 用途 |
|-----|-----------|------|
| ゲームエンジン本体 | 4MB | エンジンコード・スタック |
| グラフィックバッファ | 6MB | フレームバッファ・Zバッファ・VRAM共有 |
| キャラクターモデル | 4MB | プレイヤー+画面内敵（最大6体分） |
| ステージデータ | 8MB | 現在エリアのモデル・テクスチャ |
| サウンドバッファ | 4MB | BGMストリーミング+SE（16チャンネル分） |
| スクリプト・テキスト | 2MB | シナリオテキスト・AIスクリプト |
| その他（スタック・ヒープ） | 4MB | 動的確保用（断片化対策のプール管理） |
| **合計** | **32MB** | |

### メモリ管理ルール
1. 動的malloc/freeは禁止。すべてメモリプールから確保する
2. エリア遷移時にエリアデータをUMDから再ロード（非同期ローディング使用）
3. キャラクターモデルはLRU（最近最も使用されていないものを破棄）キャッシュで管理
4. テクスチャ圧縮を必ず適用（4bit Palettedで75%圧縮）

## AI実装方針

### 敵AIの状態機械

```
[Patrol（巡回）]
    ↓ プレイヤー発見（視野内に入る）
[Alert（警戒）]
    ↓ 接近または攻撃を受ける
[Combat（戦闘）]
    ↓ HP 20%以下
[Desperate（必死）] → 超気合技使用
    ↓ HP 0
[Defeated（撃破）]
```

### AIパラメータ
- 視野距離：通常敵8m、ボス10m
- 視野角：120度（正面の広角範囲）
- 攻撃間隔：キャラクターごとに設定（通常：1.5〜2.5秒）
- ガード確率：一般：10%、中ボス：30%、ボス：50%
- 超気合技使用トリガー：HP 25%以下かつ主人公が近距離にいる場合

## サウンドシステム

### BGM
- フォーマット：AT3（ATRAC3）独自ストリーミング
- サンプリングレート：44.1kHz / 16bit ステレオ
- ビットレート：64kbps（容量節約）
- ストリーミング方式：UMDから非同期ロード、バッファ2秒分先読み

### SE（サウンドエフェクト）
- フォーマット：VAG（PS独自フォーマット）
- サンプリングレート：22.05kHz / 16bit モノラル（空間効果はソフトウェアで付与）
- 同時発音数：最大16チャンネル
- SEバンク：シーン単位でメモリにロード

### ボイス
- メインキャラクターのタンカ・気合い叫びに日本語フルボイスを実装
- ザコキャラクターはセリフのみ（ボイスなし）
- ボイスデータはAT3形式で圧縮（16kbps）

## ロード最適化

### UMDロード設計
- エリア遷移時のロード時間目標：10秒以内
- 非同期ストリーミングローダーを実装し、エリア遷移アニメーション中にバックグラウンドでロード
- よく使うアセット（共通キャラモデル・UIなど）はゲーム起動時に一括ロードしてRAMに常駐させる

### シームレスロード
- フィールド内の同一エリアではロードなしで移動可能
- エリアをまたぐ場合のみ短いトランジション（扉・フェードアウト）でロード時間を隠蔽

### プリロード戦略
- 現在エリアの隣接エリアデータをバックグラウンドで先読みしておく（デュアルコア活用）
- PSP-2000以降（64MB RAM）では先読みデータを多めにロードして待ち時間をさらに短縮
