# セーブデータ仕様書 — 喧嘩番長5 漢の法則

## セーブデータ構造定義

### ファイル情報
- ファイル名：`KNK5_SAVE_SLOTx.bin`（x = スロット番号 0〜2）
- フォーマット：バイナリ形式（リトルエンディアン）
- チェックサム：CRC32（4byte）をファイル末尾に付加
- 総ファイルサイズ：推定512KB〜1MB（スロット1個あたり）

### ヘッダー構造（先頭64byte）

| フィールド名 | 型 | サイズ | 説明 |
|---|---|---|---|
| magic | BYTE[4] | 4byte | マジックナンバー「KNK5」 |
| version | UINT16 | 2byte | セーブデータバージョン |
| slot_id | UINT8 | 1byte | スロット番号（0〜2） |
| playtime_sec | UINT32 | 4byte | 累計プレイ時間（秒） |
| save_chapter | UINT8 | 1byte | セーブ時の章番号 |
| save_stage_id | UINT8 | 1byte | セーブ時のステージID |
| timestamp | UINT32 | 4byte | セーブ日時（Unix時間） |
| thumbnail_offset | UINT32 | 4byte | サムネイル画像オフセット |
| reserved | BYTE[42] | 42byte | 将来の拡張用予備領域 |

## 保存タイミング

### 自動セーブ
- 各章のボス撃破後に自動セーブ確認ダイアログを表示
- 外伝シナリオクリア時に自動セーブ確認
- ゲーム終了ボタン押下時（ホームメニューへ）に自動セーブ確認

### 手動セーブ
- バトル結果画面でセーブ確認
- STARTボタン→ポーズメニュー→「セーブする」で任意タイミングでセーブ
- フィールドの特定ポイント（セーブポイントアイコン付近）で手動セーブ

## ストレージ使用量

| 項目 | サイズ |
|---|---|
| セーブデータ1スロット | 約512KB〜1MB |
| セーブスロット3スロット合計 | 約1.5〜3MB |
| サムネイル画像（JPEG・スロットごと） | 約30KB × 3 = 90KB |
| 合計推定 | 約2〜3.5MB |

---

## セーブスロット数

**セーブスロット：3スロット**（スロット0・1・2）

各スロットには以下の情報がサムネイル画面で確認可能：
- 主人公名・所属高校名
- 現在のプレイ章
- 累計プレイ時間
- セーブ日時
- ダチ数・縄張り制覇率

セーブスロットが埋まっている場合、上書き確認ダイアログを表示する。セーブデータの削除はメインメニューの「データ管理」から行う。
