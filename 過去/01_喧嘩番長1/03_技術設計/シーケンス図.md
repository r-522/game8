# シーケンス図 — 喧嘩番長

## 1. フィールドでの遭遇〜メンチ開始

```mermaid
sequenceDiagram
    participant P as Player（プレイヤー操作）
    participant FM as FieldManager
    participant E as Enemy（敵）
    participant MS as MenchiSystem
    participant OS as OtokogiSystem
    participant UI as UI/HUD

    P->>FM: フィールド移動（スティック入力）
    FM->>E: 敵との距離チェック（毎フレーム）
    E-->>FM: 距離が遭遇範囲内
    FM->>UI: 敵インジケーター表示（黄色アイコン）

    P->>MS: Rボタン長押し（メンチ起動）
    MS->>UI: メンチビームエフェクト開始
    MS->>OS: 気合ゲージ消費チェック
    OS-->>MS: 消費量確認・残気合返却

    alt 気合十分かつシブイ/普通状態
        MS->>E: メンチビーム照射（ロックオン）
        E-->>MS: メンチ返し（ビーム衝突演出）
        MS->>UI: メンチゲージ表示・上昇
        P->>MS: ○ボタン（ケンカ成立）
        MS->>FM: メンチ成立通知
        FM->>TankaSystem: タンカフェーズ開始要請
    else シャバゾウ状態
        MS->>UI: 「メンチビームが弱い…」メッセージ
        MS->>P: メンチ不成立（キャンセル）
    else 気合不足
        MS->>UI: 「気合が足りない！」メッセージ
        MS->>P: メンチ不発
    end
```

## 2. メンチ勝利〜タンカ開始

```mermaid
sequenceDiagram
    participant MS as MenchiSystem
    participant TS as TankaSystem
    participant DB as TankaDatabase
    participant P as Player
    participant E as Enemy
    participant UI as UI/HUD
    participant OS as OtokogiSystem

    MS->>TS: タンカ開始要請（メンチ成立通知）
    TS->>DB: タンカスクリプト取得（対象キャラID指定）
    DB-->>TS: TankaScript（セリフ断片・正解ボタン配列）

    TS->>UI: タンカ画面表示
    TS->>UI: セリフ断片をバラバラに配置表示
    TS->>UI: 制限時間タイマー開始

    loop タイマー残時間 > 0
        P->>TS: ボタン入力
        TS->>TS: 入力順序チェック
        alt 正解入力
            TS->>UI: 入力成功フィードバック（光るエフェクト）
            TS->>TS: 次の入力待ち
        else 不正解入力
            TS->>UI: 失敗エフェクト
            TS->>TS: タンカ失敗確定
            break
        end
    end

    alt 全入力成功（タンカ成功）
        TS->>OS: 男気上昇(+5)
        TS->>UI: 「タンカ成功！先制攻撃ボーナス！」表示
        TS->>CombatSystem: 戦闘開始（先制攻撃ボーナスフラグ付き）
    else 裏正解入力成功
        TS->>OS: 男気上昇(+8)
        TS->>UI: 「会心のタンカ！！」特殊演出
        TS->>CombatSystem: 戦闘開始（超強化先制ボーナス）
    else タンカ失敗 or タイムアウト
        TS->>OS: 男気低下(-5)
        TS->>UI: 「タンカ失敗…先制を許した」表示
        TS->>CombatSystem: 戦闘開始（敵先制攻撃フラグ付き）
    end
```

## 3. タンカ結果〜戦闘開始

```mermaid
sequenceDiagram
    participant TS as TankaSystem
    participant CS as CombatSystem
    participant P as Player
    participant E as Enemy
    participant HB as HitboxSystem
    participant UI as UI/HUD
    participant SND as SoundSystem

    TS->>CS: 戦闘開始（先制ボーナス/ペナルティフラグ）
    CS->>UI: 戦闘画面へ遷移（カメラ切り替え）
    SND->>SND: 戦闘BGM再生開始

    alt 先制攻撃ボーナスあり
        CS->>P: 先制攻撃アニメーション再生
        CS->>HB: プレイヤーヒットボックスをアクティブ化
        HB->>CS: ヒット判定（E座標との交差チェック）
        CS->>E: ダメージ計算・適用（×1.5補正）
        CS->>UI: コンボカウンター更新
    else 敵先制攻撃あり
        CS->>E: 敵先制攻撃アニメーション
        CS->>HB: 敵ヒットボックスをアクティブ化
        HB->>CS: ヒット判定
        CS->>P: ダメージ適用
        CS->>UI: 体力ゲージ更新
    end

    CS->>CS: 通常戦闘フェーズへ移行
    note over CS: 毎フレーム入力処理・AI処理・ヒット判定ループ
```

## 4. 戦闘終了〜男気更新〜結果画面

```mermaid
sequenceDiagram
    participant CS as CombatSystem
    participant P as Player
    participant E as Enemy
    participant OS as OtokogiSystem
    participant SS as SaveSystem
    participant UI as UI/HUD
    participant FM as FieldManager
    participant SND as SoundSystem

    alt プレイヤーがKO（HP=0）
        CS->>UI: ゲームオーバー演出表示
        CS->>SND: ゲームオーバーSE再生
        UI->>P: コンティニュー確認ダイアログ
        alt コンティニュー選択
            P->>SS: 最終セーブデータロード
            SS->>FM: 自宅前からリスタート
        else タイトルへ
            FM->>FM: タイトル画面へ遷移
        end
    else 敵がKO（HP=0）
        CS->>UI: 勝利演出（KOエフェクト・停止演出）
        CS->>SND: 勝利BGM/SE再生
        CS->>OS: 男気更新処理
        OS->>OS: 勝利による男気上昇(+3〜+10)
        OS->>UI: 男気メーター更新アニメーション
        CS->>P: 喧嘩慣れ度加算
        P->>P: 番長度チェック（レベルアップ条件確認）

        alt 番長度アップ条件達成
            P->>UI: レベルアップ演出表示
            P->>P: パラメータ上昇・新技解放
        end

        CS->>UI: 結果画面表示（獲得EXP・男気変化・アイテム）
        UI->>FM: フィールドへ戻る

        alt ボス撃破
            FM->>UI: ストーリーイベント/ムービー再生
            FM->>FM: 次ステージアンロック処理
        end
    end
```

## シーケンス処理参加者の役割まとめ

| 参加者 | 役割 |
|--------|------|
| Player | プレイヤーの入力・パラメータ保持 |
| Enemy | 敵のAI・行動・パラメータ |
| FieldManager | フィールド全体の管理・遷移制御 |
| MenchiSystem | メンチビーム視線対決のロジック |
| TankaSystem | タンカ入力ゲームのロジック |
| CombatSystem | 戦闘全体の制御・ダメージ計算 |
| HitboxSystem | ヒットボックス衝突判定 |
| OtokogiSystem | 男気パラメータの増減管理 |
| SaveSystem | セーブ・ロード処理 |
| UI/HUD | 画面表示・エフェクト |
| SoundSystem | BGM・SE・ボイスの再生管理 |
