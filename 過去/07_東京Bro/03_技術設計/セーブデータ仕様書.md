# セーブデータ仕様書 — 東京Bro（喧嘩番長-Crash Battle-）

## セーブデータ構造定義

本作はスマートフォンアプリのため、セーブデータはクラウドサーバー（スパイク・チュンソフト管理サーバー）に自動保存される。端末ローカルにはキャッシュとして一部データを保持するが、正式なデータはサーバーに存在する。

### ルートデータ構造

| フィールド名 | 型 | サイズ | 説明 |
|------------|-----|--------|------|
| player_id | string | 36byte | プレイヤー固有UUID（サーバー発行） |
| player_name | string | 最大16文字 | プレイヤー設定名 |
| player_rank | int | 4byte | プレイヤーランク（バトル経験値累計で決定） |
| player_exp | int | 4byte | 現在の累計プレイヤー経験値 |
| stamina | int | 4byte | 現在のスタミナ残量 |
| stamina_last_recovery_time | timestamp | 8byte | スタミナ最終回復時刻（Unix時間） |
| kousho | int | 4byte | 所持校章数（ゲーム内通貨） |
| otokogi_value | int | 4byte | 男気ゲージ現在値（0〜1000） |
| story_chapter | int | 4byte | ストーリー進行章番号（1〜5） |
| created_at | timestamp | 8byte | アカウント作成日時 |
| updated_at | timestamp | 8byte | 最終更新日時 |

### キャラクターデータ構造（per character）

| フィールド名 | 型 | サイズ | 説明 |
|------------|-----|--------|------|
| user_character_id | string | 36byte | ユーザー所持キャラ固有ID |
| master_character_id | int | 4byte | キャラクターマスターへの参照 |
| level | int | 4byte | 現在レベル |
| current_exp | int | 4byte | 現在の累計経験値 |
| rarity | int | 4byte | 現在の★数（1〜6） |
| skill_level | int | 4byte | 超気合技スキルLv（1〜10） |
| acquired_at | timestamp | 8byte | 入手日時 |
| is_favorite | bool | 1byte | お気に入りフラグ（合成素材候補から除外） |

### デッキ編成データ

| フィールド名 | 型 | サイズ | 説明 |
|------------|-----|--------|------|
| deck_id | int | 4byte | デッキ番号（1〜5） |
| leader_character_id | string | 36byte | リーダーキャラのuser_character_id |
| member_character_ids | string[] | 36byte×5 | メンバーキャラID配列（最大5枠） |

## 保存タイミング

### 自動保存
| タイミング | 保存内容 |
|-----------|---------|
| バトル終了時 | ステージクリア状態・報酬・経験値・男気値 |
| キャラ合成・進化時 | キャラクターデータ |
| デッキ変更時 | デッキ編成データ |
| ガチャ実行時 | 所持キャラ・校章残高 |
| アプリ非アクティブ化時（バックグラウンド移行）| 全データの差分をサーバーへPUSH |
| 定期自動同期（5分毎） | スタミナ残量・校章残高 |

### 手動保存
- 本作はサーバー型セーブのため、プレイヤーによる明示的な手動保存操作は存在しない
- 設定画面の「データ引継ぎ」機能でバックアップコードを発行可能

## ストレージ使用量

| ストレージ区分 | 推定使用量 |
|--------------|-----------|
| アプリ本体（インストール時） | 約120〜200MB |
| ローカルキャッシュ（マスターデータ） | 約5〜15MB |
| ローカルキャッシュ（画像・テクスチャ） | 約30〜80MB |
| サーバー上のセーブデータ | 約50KB〜1MB（キャラ所持数依存） |

## セーブスロット数

本作はサーバー型アカウントセーブのため、セーブスロットの概念は存在しない。
1アカウントに1つのセーブデータが紐づく。

- **機種変更対応**：ゲーム内の「データ引継ぎ」機能でバックアップコードを発行し、新端末でコードを入力することでデータを移行可能。
- **複数端末対応**：同一アカウントで複数端末からのアクセスは想定しないが、サーバー上のデータが常に最新として扱われる。
- **サービス終了後**：サービス終了時点でサーバーへのアクセスが不可能になり、データへのアクセスも失われる。
