# 通信プロトコル仕様書 — 喧嘩番長7「波嵐烈伝」

## 通信アーキテクチャ

```
Browser (Client)
    │
    ├── Supabase Browser Client（ANON_KEY）
    │     ├── READ: マスターデータ取得
    │     ├── READ/WRITE: プレイヤーデータ（RLS適用）
    │     └── Auth: 匿名認証
    │
    └── Next.js Server Actions（SERVICE_ROLE_KEY）
          ├── ランキングスコア検証・書き込み
          ├── セーブデータ書き込み検証
          └── 実績解放（重複チェック）
```

## Supabase API

### 認証

#### 匿名ログイン
```typescript
// client.ts
const supabase = createClient()
const { data, error } = await supabase.auth.signInAnonymously()
// data.user.id → kenka_players.idとして使用
```

#### セッション確認
```typescript
const { data: { session } } = await supabase.auth.getSession()
if (!session) {
  await supabase.auth.signInAnonymously()
}
```

### マスターデータ取得（全ユーザー読み取り可）

#### スキル一覧
```typescript
const { data: skills } = await supabase
  .from('kenka_skills')
  .select('*')
  .order('id')
```

#### ステージ情報
```typescript
const { data: stage } = await supabase
  .from('kenka_stages')
  .select('*')
  .eq('id', stageId)
  .single()
```


## Server Actions エンドポイント

### submitRankingScore

**エンドポイント**: `app/actions.ts` → `submitRankingScore`  
**認証**: Supabase Server Client（service_role）  
**用途**: スコア改ざん防止のためサーバーサイドで検証

```typescript
// リクエスト
interface SubmitRankingScoreInput {
  score: number   // 0〜999999999
  mode: string    // 'total_score' | 'combo' | 'clear_time'
}

// レスポンス
interface SubmitRankingScoreResult {
  success: boolean
  error?: string
  isNewRecord?: boolean
}
```

**バリデーション**:
- `score < 0` → エラー
- `score > 999999999` → エラー
- `mode` が許容値以外 → エラー
- 既存スコアより低い場合 → 更新しない（`success: false`ではなく`isNewRecord: false`）

## エラーハンドリング方針

| エラー種別 | 対処 |
|----------|------|
| 認証切れ | 自動再認証（signInAnonymously再実行） |
| ネットワークエラー | ローカル状態は維持、次回セーブ時に再試行 |
| RLS違反 | エラーログのみ、ユーザーには「セーブに失敗しました」 |
| バリデーション違反 | エラーメッセージを表示、データ変更なし |
| Supabase障害 | オフラインモードで継続、再接続時に同期 |

## レート制限・セキュリティ

| 操作 | 制限 |
|------|-----|
| ランキング書き込み | 10秒に1回まで（Server Action内でチェック） |
| セーブ | 30秒に1回まで（UI側でデバウンス） |
| 認証 | Supabaseデフォルト制限に準拠 |

### 環境変数
```
NEXT_PUBLIC_SUPABASE_URL    → フロントエンド接続先（公開可）
NEXT_PUBLIC_SUPABASE_ANON_KEY → フロントエンド認証（公開可・RLSで制御）
SUPABASE_SERVICE_ROLE_KEY   → Server Actionsのみ（絶対に公開しない）
```
