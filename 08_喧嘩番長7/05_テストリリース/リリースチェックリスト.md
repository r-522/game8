## Supabase初期設定手順

### 1. プロジェクト作成
1. `supabase.com` → New Project
2. プロジェクト名: `kenka-bancho-7`
3. データベースパスワード: 安全なパスワードを設定
4. リージョン: Northeast Asia（東京）

### 2. マイグレーション実行
```bash
# Supabase CLI使用
supabase login
supabase link --project-ref YOUR_PROJECT_ID
supabase db push

# または SQL Editorから直接実行
# supabase/migrations/0001_init.sql の内容をペースト
```

### 3. シードデータ投入
```bash
# SQL Editorから
# supabase/seed.sql の内容をペースト・実行
```

### 4. 匿名認証有効化
1. Supabase Dashboard → Authentication → Providers
2. Anonymous → Enable

### 5. APIキー取得
1. Settings → API
2. `URL` → `NEXT_PUBLIC_SUPABASE_URL`
3. `anon/public` → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. `service_role` → `SUPABASE_SERVICE_ROLE_KEY`（絶対に公開しない）

## 緊急対応手順

### Supabase障害時
- ゲームは起動するが、セーブ・ランキングが機能しない状態になる
- `console.error`でユーザーに通知（「オンライン機能が一時的に使えません」）
- 復旧後、次回ゲームスタート時に自動再接続

### ビルドエラー時のロールバック
```bash
# Vercelダッシュボード → Deployments → 直前の成功デプロイ → Redeploy
```

### 緊急連絡先
- Supabase Status: https://status.supabase.com
- Vercel Status: https://www.vercel-status.com
